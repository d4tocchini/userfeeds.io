01. Introductions

The Userfeeds protocol works as a set of relevance layers:

Evaluation Layer

The first layer of the cycle where private key holders sign and publish off-chain messages - "proofs-of-evaluation" - with their evaluations of arbitrary information.

"Proof-Of-Evaluation" Message Type

Proofs-of-Evaluation is a basic signal of information relevance in the Userfeeds Protocol. Using "proofs-of-evaluation" private key holders can create new relationships between their blockchain addresses and arbitrary information. Think links, likes, upvotes, downvotes, transactions etc.

Publication, Aggregation and Ranking Layers

'Proofs-of-evaluation' are published, aggregated and then ranked using ranking algorithms. The output of ranking algorithms is passed to the application interfaces at the Display Layer.

Userfeeds Relevance Platform

The Userfeeds Relevance Platform is a first working implementation of the Userfeeds Protocol that aggregates 'proofs-of-evaluations' from the Ethereum blockchain and powers ranking algorithms running on top of the aggregated graph.

See example.

Display Layer

The Display Layer is the last layer in the cycle. Application interfaces present the output of the algorithms to the end-user. The user action kickstarts a new evaluation cycle at the Evaluation layer.

Interfaces & Applications

The first application built on top of the Userfeeds Protocol is LinkExchange.io which we call "Google AdSense For Tokens".

03. Possibilities

Reputation Currencies

Our model enables quick creation of reputation currency schemes. You can reward users for actions taken within your application with your own currency.

Decentralized Voting Systems

Governance is still an unsolved problem in the cryptocurrency world. With Userfeeds you can design voting systems to quickly gather feedback on important topics.

Blockchain-based Ad Networks

Experiment with the blockchain-based equivalent of Google Adsense where you can literally look behind the banner and see the the identity, reputation and history of the advertiser.

Decentralized Reddit/Twitter

Allow your token-based community to create their own view of social media destinations. Let your community curate Twitter/Reddit/Facebook to surface topics most relevant to them.
